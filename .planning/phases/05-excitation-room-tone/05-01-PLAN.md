**ExcitationSection.cpp** -- Full implementation:

`prepare()`:
- Store sampleRate, blockSize
- Adaptive oversampling: sampleRate <= 50000 -> order 2 (4x); <= 100000 -> order 1 (2x); else -> order 0 (1x)
- Create `juce::dsp::Oversampling<float>` with 2 channels, the computed order, `filterHalfBandFIREquiripple`, `true` (isMaxQuality), `true` (useIntegerLatency)
- Call `oversampler->initProcessing(blockSize)`
- Store `latencySamples = (int)oversampler->getLatencyInSamples()`
- Set up LR crossover filters: for each of the 4 filters, call `setCutoffFrequency(freq)` and `setType(...)`. Low pair: LP/HP at kLowCrossFreq. High pair: LP/HP at kHighCrossFreq. Use `juce::dsp::LinkwitzRileyFilterType::lowpass` / `highpass`.
- Allocate work buffers (lowBand, midBand, highBand, dryCopy) sized to 2 channels x blockSize
- Initialize DC blocker IIR filters with HPF at 5Hz (use `juce::dsp::IIR::Coefficients<float>::makeHighPass(sampleRate, 5.0f)`)
- Reset SmoothedValues: `driveSmoothed.reset(sampleRate, 0.025)`, `bypassBlend.reset(sampleRate, 0.010)`
- Set initial values: `driveSmoothed.setCurrentAndTargetValue(currentDrive)`, `bypassBlend.setCurrentAndTargetValue(0.0f)` (default: not bypassed, so blend=1.0 means fully wet -- but match the AirSection convention)

For bypass blend convention: Match AirSection. When bypassed, blend target = 0.0 (fully dry). When not bypassed, blend target = 1.0 (fully processed). In process: `output = dry*(1-blend) + wet*blend`.

`process()`:
1. Get numSamples, numChannels. Early exit if numChannels < 1 or numSamples < 1.
2. Check bypass: if `!bypassBlend.isSmoothing() && bypassBlend.getTargetValue() <= 0.0f` -> return (fully bypassed, skip all DSP).
3. Update drive params from current values: call `updateDriveParams()`.
4. Copy input to dryCopy buffer for bypass crossfade.
5. **Single-oversampler approach** (per research recommendation): Upsample the full stereo buffer, split into 3 bands at oversampled rate, waveshape each band, recombine, downsample.

Detailed process flow:
```
a. Create AudioBlock from buffer
b. auto oversampledBlock = oversampler->processSamplesUp(block)
c. For each oversampled sample:
   - Read L/R from oversampledBlock
   - Split through LR crossovers per-sample:
     lowL = lowCrossLP.processSample(0, sampleL)
     midHighL = lowCrossHP.processSample(0, sampleL)
     midL = highCrossLP.processSample(0, midHighL)
     highL = highCrossHP.processSample(0, midHighL)
     (same for R with channel 1)
   - Waveshape each band with smoothed G values:
     For low/mid/high bands, apply: tanh(sample * G) / tanh(G)
     Guard against G < 0.001 (near-zero G: output = sample, linear passthrough)
   - Recombine: outputL = lowL + midL + highL (same for R)
   - Write back to oversampledBlock
d. oversampler->processSamplesDown(block)  -- downsamples back into buffer
```

Note: The `driveSmoothed` SmoothedValue operates at the base sample rate (read once per base-rate sample or once per block), not at the oversampled rate. Simplest approach: read smoothed drive value once per block, compute G values once per block. This avoids the complexity of per-oversampled-sample smoothing and is fine because Drive changes are slow (knob movements).

6. Apply DC blocker to L and R channels (at base rate, post-downsample).
7. Bypass crossfade: for each sample, `blend = bypassBlend.getNextValue()`, `output = dry*(1-blend) + wet*blend`.

`reset()`:
- Reset oversampler: `oversampler->reset()`
- Reset all LR crossover filters
- Reset DC blockers
- Reset SmoothedValues to current values

`setBypass()`:
- Set `bypassBlend.setTargetValue(shouldBypass ? 0.0f : 1.0f)`

`setDrive()`, `setMaterial()`, `setRoomSize()`:
- Store values in member variables. `setDrive` also sets `driveSmoothed.setTargetValue(drive)`.

`getLatencySamples()`:
- Return `latencySamples` member.

CRITICAL: No memory allocations in process(). All buffers pre-allocated in prepare(). The oversampler manages its own internal buffers.

IMPORTANT: The tanh waveshaper must be SYMMETRIC (no DC bias). This models room excitation, not gear distortion. Do NOT add any asymmetric component.

IMPORTANT: The LinkwitzRileyFilter in JUCE requires calling `prepare()` with a ProcessSpec before use. Do this in `prepare()` for all 4 filter instances.
