**Source/Parameters.h** -- Central parameter definition file. Contains:

1. `namespace ParamIDs` with `inline constexpr auto` for all 20 parameter IDs using stage-prefixed snake_case:
   - Stage I (Cabinet): `cab_body`, `cab_type`, `cab_bypass`
   - Stage II (Reflections): `refl_size`, `refl_shape`, `refl_prox`, `refl_bypass`
   - Stage III (Air): `air_amount`, `air_char`, `air_bypass`
   - Stage IV (Excitation): `excit_drive`, `excit_bypass`
   - Stage V (Room Tone): `tone_amb`, `tone_bypass`
   - Stage VI (Diffuse Tail): `tail_decay`, `tail_diff`, `tail_bypass`
   - Output: `out_mix`, `out_level`

2. `juce::AudioProcessorValueTreeState::ParameterLayout createParameterLayout()` function that registers all 20 parameters with these exact specifications from the design handoff:

   **Float parameters (12):**
   - `cab_body`: 0.0-1.0, default 0.5, display "I Cabinet - Body", percent format
   - `refl_size`: 0.0-1.0, default 0.4, display "II Reflections - Room Size", text labels (Small/Sm-Med/Medium/Med-Lg/Large) using withStringFromValueFunction
   - `refl_shape`: 0.0-1.0, default 0.4, display "II Reflections - Shape", text labels (Regular/Reg-Irr/Mixed/Irr-Reg/Irregular)
   - `refl_prox`: 0.0-1.0, default 0.3, display "II Reflections - Proximity", text labels (Near/Nr-Mid/Middle/Mid-Far/Far)
   - `air_amount`: 0.0-1.0, default 0.4, display "III Air - Amount", percent format
   - `excit_drive`: 0.0-1.0, default 0.25, display "IV Excitation - Drive", percent format
   - `tone_amb`: 0.0-1.0, default 0.1, display "V Room Tone - Ambience", percent format
   - `tail_decay`: 50.0-500.0, default 150.0, display "VI Tail - Decay", milliseconds suffix " ms"
   - `tail_diff`: 0.0-1.0, default 0.6, display "VI Tail - Diffusion", percent format
   - `out_mix`: 0.0-1.0 (representing 0-100%), default 0.7, display "Output - Mix", percent format
   - `out_level`: -24.0 to 6.0, default 0.0, display "Output - Level", dB suffix " dB"

   Use step size 0.01 for 0-1 range floats, 1.0 for tail_decay, 0.1 for out_level.

   **Choice parameters (2):**
   - `cab_type`: choices ["Open", "Closed", "Combo"], default index 1 (Closed), display "I Cabinet - Type"
   - `air_char`: choices ["Warm", "Neutral"], default index 0 (Warm), display "III Air - Character"

   **Bool parameters (6):**
   - `cab_bypass`: default false, display "I Cabinet - Bypass"
   - `refl_bypass`: default false, display "II Reflections - Bypass"
   - `air_bypass`: default false, display "III Air - Bypass"
   - `excit_bypass`: default false, display "IV Excitation - Bypass"
   - `tone_bypass`: default true (Room Tone bypassed by default per design handoff), display "V Room Tone - Bypass"
   - `tail_bypass`: default false, display "VI Tail - Bypass"

   All parameters use ParameterID version 1: `juce::ParameterID { "param_id", 1 }`.
