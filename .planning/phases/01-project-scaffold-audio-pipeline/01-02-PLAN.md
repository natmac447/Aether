**6 DSP Stage Placeholder Classes (CabinetSection, ReflectionsSection, AirSection, ExcitationSection, RoomToneSection, DiffuseTailSection):**

Each is a concrete class (no virtual base -- matches Crucible pattern, avoids virtual dispatch on audio thread). Each follows the same interface by convention:

```cpp
class XxxSection
{
public:
    void prepare(double sampleRate, int samplesPerBlock);
    void process(juce::AudioBuffer<float>& buffer);
    void reset();
    void setBypass(bool bypassed);
private:
    bool bypassed = false;
    double currentSampleRate = 44100.0;
};
```

- `prepare()`: Store sampleRate and samplesPerBlock. Add a comment describing what this stage will do in its target phase (e.g., "Phase 2: Short FDN for cabinet body resonance").
- `process()`: When bypassed, return immediately (no-op). When not bypassed, also return immediately (placeholder -- buffer passes through unchanged since processing is in-place). Add a comment: "// TODO Phase N: Implement XYZ".
- `reset()`: Clear any future state. No-op for now.
- `setBypass()`: Set the bypassed flag.

Each stage stores `currentSampleRate` to satisfy ENG-02 (sample-rate awareness pattern established for later phases).

**OutputSection (from Crucible pattern -- real implementation):**

```cpp
class OutputSection
{
public:
    void prepare(double sampleRate, int samplesPerBlock);
    void process(juce::AudioBuffer<float>& buffer, float gainDb);
    void reset();
private:
    juce::SmoothedValue<float, juce::ValueSmoothingTypes::Multiplicative> gainSmoothed;
};
```

- `prepare()`: `gainSmoothed.reset(sampleRate, 0.020)` (20ms ramp for ENG-01). `gainSmoothed.setCurrentAndTargetValue(1.0f)`.
- `process(buffer, gainDb)`: Convert gainDb to linear using `juce::Decibels::decibelsToGain(gainDb, -100.0f)`. Set target value. If smoothing, process sample-by-sample. If not smoothing, use `buffer.applyGain()`. This is the exact Crucible OutputSection pattern from the research.
- `reset()`: `gainSmoothed.setCurrentAndTargetValue(1.0f)`.

**MixSection (DryWetMixer + auto-gain compensation):**

```cpp
class MixSection
{
public:
    void prepare(double sampleRate, int samplesPerBlock);
    void pushDrySamples(const juce::dsp::AudioBlock<float>& dryBlock);
    void mixWetSamples(juce::dsp::AudioBlock<float>& wetBlock);
    void setMixLevel(float mixValue);
    void applyAutoGainCompensation(juce::AudioBuffer<float>& buffer, float mixValue);
    void reset();
private:
    juce::dsp::DryWetMixer<float> dryWetMixer;
    juce::SmoothedValue<float, juce::ValueSmoothingTypes::Multiplicative> compensationSmoothed;
    float currentMix = 0.7f;
};
```

- `prepare()`: Initialize DryWetMixer with `juce::dsp::ProcessSpec { sampleRate, (uint32) samplesPerBlock, 2 }`. Set mixing rule to `juce::dsp::DryWetMixingRule::sin3dB` (equal-power crossfade, standard for reverb). Set wet latency to 0 (no DSP latency in Phase 1). Initialize `compensationSmoothed.reset(sampleRate, 0.020)` with current and target value 1.0f.
- `pushDrySamples()`: Forward to `dryWetMixer.pushDrySamples(dryBlock)`. Called at the top of processBlock to capture pre-processing dry signal (OUT-04).
- `setMixLevel()`: Call `dryWetMixer.setWetMixProportion(mixValue)`. Store for auto-gain.
- `mixWetSamples()`: Forward to `dryWetMixer.mixWetSamples(wetBlock)`. Called after all 6 stages.
- `applyAutoGainCompensation()`: Implement the auto-gain curve from research: `compensationDb = -2.5f * std::pow(mixValue, 1.5f)`. Convert to gain, smooth, apply sample-by-sample if smoothing or `buffer.applyGain()` if settled. At Mix=0% = 0dB compensation, Mix=50% = -0.88dB, Mix=100% = -2.5dB. This satisfies OUT-03.
- `reset()`: Reset DryWetMixer and SmoothedValue.

**Important real-time safety for all sections (ENG-05):**
- No heap allocations in any process() method
- All SmoothedValue instances pre-initialized in prepare()
- No juce::String construction in process paths

**PluginProcessor.cpp -- processBlock():**
Implement the complete signal flow per ENG-06: Input -> I -> II -> III -> IV -> V -> VI -> Mix -> Output.

```
void AetherProcessor::processBlock(juce::AudioBuffer<float>& buffer, juce::MidiBuffer& midiMessages)
{
    juce::ignoreUnused(midiMessages);
    juce::ScopedNoDenormals noDenormals;  // ENG-05: prevent denormal CPU spikes

    // Clear unused output channels (ENG-04: works with any channel config)
    for (auto i = getTotalNumInputChannels(); i < getTotalNumOutputChannels(); ++i)
        buffer.clear(i, 0, buffer.getNumSamples());

    // Read parameters and push to sections
    updateStageParams();

    // OUT-04: Capture dry signal at input (pre all processing)
    mixSection.pushDrySamples(juce::dsp::AudioBlock<float>(buffer));

    // ENG-06: Fixed serial chain I -> II -> III -> IV -> V -> VI
    cabinetSection.process(buffer);
    reflectionsSection.process(buffer);
    airSection.process(buffer);
    excitationSection.process(buffer);
    roomToneSection.process(buffer);
    diffuseTailSection.process(buffer);

    // OUT-01: Mix dry/wet with equal-power crossfade
    {
        juce::dsp::AudioBlock<float> wetBlock(buffer);
        mixSection.mixWetSamples(wetBlock);
    }

    // OUT-03: Auto-gain compensation
    float currentMix = outMixParam->load();
    mixSection.applyAutoGainCompensation(buffer, currentMix);

    // OUT-02: Output level trim
    outputSection.process(buffer, outLevelParam->load());
}
```
