  <action>
    Create AetherKnob -- a juce::Component that owns a juce::Slider (RotaryHorizontalVerticalDrag), a value display juce::Label, and a knob name label. Implements Slider::Listener and Timer for animation.

    **Constructor parameters:**
    - `const juce::String& name` -- knob label text (e.g., "BODY", "AIR")
    - `int knobSize` -- 56 (standard) or 64 (center room controls), default 56
    - Optional: `std::function<juce::String(float)> valueFormatter` for custom value display (e.g., "62% (Medium)")

    **Slider configuration (in constructor):**
    - `slider.setSliderStyle(juce::Slider::RotaryHorizontalVerticalDrag)`
    - `slider.setTextBoxStyle(juce::Slider::NoTextBox, false, 0, 0)` -- we draw our own value display
    - `slider.setRotaryParameters(pi*1.25f, pi*2.75f, true)` -- 270 degree arc, 7 to 5 o'clock
    - `slider.setScrollWheelEnabled(true)`
    - `slider.setDoubleClickReturnValue(true, 0.5f)` -- will be updated per-param by attachToParameter

    **Drag sensitivity (LOCKED DECISION: ~300px for full sweep):**
    - Use velocity mode: `slider.setVelocityBasedMode(true)`
    - `slider.setVelocityModeParameters(0.5, 1, 0.0, true, juce::ModifierKeys::shiftModifier)`
    - The sensitivity 0.5 means user needs ~double the default pixels for full range (~300px instead of ~150px). Shift key activates fine mode (10x slower via JUCE's default fine control handling).
    - `slider.setSliderSnapsToMousePosition(false)` -- don't jump to click position

    **Value display label:**
    - Create a juce::Label below the knob for value readout
    - Style: EB Garamond 11px italic, Ink Faint colour, centred
    - On slider value change: update label text using valueFormatter if provided, else use parameter's string from value function
    - **Click-to-type (LOCKED DECISION):** Make the value label editable on click:
      - `valueLabel.setEditable(false, true, false)` -- single-click triggers edit
      - Implement Label::Listener::labelTextChanged: parse the user's input text, clamp to parameter range, set slider value
      - Out-of-range values clamp silently (no error message)
    - Update value label in real-time during drag via sliderValueChanged

    **Knob name label:**
    - Create a juce::Label between knob and value label for the knob name
    - Style: Spectral 9px uppercase, Ink Light colour, centred, 2px letter spacing
    - Letter spacing: Since JUCE has no letter-spacing, use the LookAndFeel's drawLabel or draw character-by-character. For simplicity, set the label text with spaces between characters or use custom paint. Recommended: override `paint()` to draw the label text char-by-char with spacing using GlyphArrangement.

    **Layout (resized()):**
    - Total component height: knobSize + 6px gap + 12px name label + 2px gap + 14px value label = knobSize + 34px
    - Total component width: max(knobSize, 70px) -- ensure value label has room
    - Knob centered at top, name label centered below, value label centered below name

    **Animation (LOCKED DECISION: smooth ~100-200ms sweep on preset load, double-click reset):**
    - Two float members: `displayValue` (what's drawn, 0-1 normalised), `targetValue` (actual param value, 0-1 normalised)
    - `isDragging` boolean flag set via Slider::Listener's sliderDragStarted/sliderDragEnded
    - In sliderValueChanged: if not dragging AND jump > 5% of range, start animation timer at 60Hz. If dragging OR small change, snap displayValue immediately.
    - In timerCallback: exponential smoothing `displayValue += 0.15f * (targetValue - displayValue)`. Stop timer when within 0.001f. Call repaint().
    - Override paint() to draw the knob indicator at `displayValue` position instead of the slider's actual position. The slider widget itself is invisible (draws nothing). All knob rendering is done in paint() by calling LookAndFeel's drawRotarySlider with displayValue-derived angle.
    - CRITICAL: The animation only affects the visual position. The Slider's actual value (and thus the parameter) jumps immediately. This prevents feedback loops with the attachment.

    **Public method: `attachToParameter(APVTS&, const String& paramID)`:**
    - Creates a SliderAttachment stored as unique_ptr member
    - After creating attachment, read the parameter's default value and call `slider.setDoubleClickReturnValue(true, defaultValue)` where defaultValue is the parameter's actual default (not 50%)
    - If the parameter has a stringFromValue function, use it in the value display

    **Value formatting for special parameters (called from parent):**
    - Provide `setValueFormatter(std::function<String(float normalised, const String& paramText)>)` method
    - For text-label params like Room Size: formatter returns "62% (Medium)" combining percentage from normalised value + descriptor from parameter's getCurrentValueAsText()
    - For Decay: formatter checks if value >= 1000ms and switches display "150ms" vs "1.2s"
    - For dB params (Output Level): formatter shows "0.0 dB" format
    - For percentage params (default): formatter returns "62%" whole numbers

    **Bypassed visual state:**
    - `setBypassed(bool)` method sets a flag. In paint(), if bypassed, draw a translucent Parchment overlay at 40% opacity over the knob area after drawing the knob. Do NOT change setAlpha or setEnabled -- controls remain interactive (LOCKED DECISION).

    Add new source files to CMakeLists.txt target_sources.

  <action>
    **AetherToggle** -- A juce::Component that displays 2 or 3 mutually exclusive options in a bordered row, synced with an AudioParameterChoice.

    **Constructor:** `AetherToggle(const juce::StringArray& options)`
    - Create one juce::TextButton per option with setClickingTogglesState(false)
    - Create a hidden juce::ComboBox for APVTS attachment bridge (addChildComponent, not visible)
    - Each button's onClick: set hiddenCombo.setSelectedItemIndex(i, sendNotification)
    - hiddenCombo.onChange: call updateButtonStates() to refresh visual toggle state
    - updateButtonStates(): For each button, setToggleState(i == selectedIndex, dontSendNotification), then repaint

    **attachToParameter(APVTS&, const String& paramID):**
    - Populate hiddenCombo with option strings (addItemList)
    - Create ComboBoxAttachment as unique_ptr member
    - Call updateButtonStates() after attachment to sync initial state

    **Layout (resized()):**
    - Arrange buttons in a horizontal row filling the component width
    - Each button gets equal width: totalWidth / numOptions
    - Height: 20px
    - No gaps between buttons -- they share borders via LookAndFeel rendering

    **Rendering (via LookAndFeel):**
    - Active option: Ink background fill, Parchment text (ink-inverted) -- handled by drawButtonBackground/drawButtonText in AetherLookAndFeel checking getToggleState()
    - Inactive options: Transparent background, Ink Faint text
    - Outer border: 1px Ink Ghost around the entire toggle row. Draw in paint() as a rounded rect border around all buttons.
    - Font: Spectral 8px uppercase, 1.5px tracking. Set button font via setLookAndFeel or override in LookAndFeel.

    **Bypassed state:**
    - `setBypassed(bool)` sets flag. In paintOverChildren(), if bypassed, draw translucent Parchment overlay at 40% opacity.

    ---

    **AetherBypassButton** -- A juce::TextButton that toggles between "In" and "Out" states, synced with an AudioParameterBool bypass parameter.

    **Constructor:** `AetherBypassButton()`
    - Set button text to "IN" initially
    - setClickingTogglesState(true)
    - onClick or onStateChange: update text to show "IN" when active (not bypassed, param false) or "OUT" when bypassed (param true)
    - Note: The bypass param is `true` when bypassed. When the button shows "IN" the bypass param is `false` (section is active). When "OUT", bypass param is `true`.

    **attachToParameter(APVTS&, const String& paramID):**
    - Create ButtonAttachment as unique_ptr member
    - After attachment, sync button text with current state

    **Visual states (via LookAndFeel + custom paint):**
    - Active ("IN"): Ink Light text, 1px Ink Light border, full opacity
    - Bypassed ("OUT"): Ink Ghost text, 1px Ink Ghost border, 60% opacity
    - Hover: opacity increases to 100% regardless of state
    - Font: Spectral 7px uppercase, 2px tracking
    - Size: ~28x16px, right-aligned in section header

    **Layout:** Fixed size, positioned by parent section component.

    **Text update logic:**
    - Listen for button state changes. The bypass parameter uses false = active, true = bypassed.
    - When ButtonAttachment syncs: getToggleState() == true means bypass is ON (show "OUT"), false means bypass is OFF (show "IN").
    - Invert display: button toggled ON = bypassed = "OUT". Button toggled OFF = active = "IN".
