Define `ShapePreset` struct (as specified in 03-RESEARCH.md Pattern 1):
```cpp
struct ShapePreset
{
    const char* name;
    const char* description;
    float delayMsL[8];   // Base delay times for L channel
    float gainL[8];       // Tap gains for L
    float panL[8];        // Pan positions (-1 to 1) for L
    float delayMsR[8];   // Base delay times for R channel
    float gainR[8];       // Tap gains for R
    float panR[8];        // Pan positions for R
    float absorptionRate; // LP cutoff reduction per ms of delay
    float tailDiffusionBias;   // For cross-stage tail influence
    float tailModalCharacter;  // For cross-stage tail influence
};
```

Class members for ReflectionsSection:
- `static constexpr int kTapsPerChannel = 8;`
- `static constexpr int kNumShapes = 7;`
- `static const ShapePreset kShapes[kNumShapes];`
- Stereo delay buffers: `signalsmith::delay::Delay<float> delayLineL, delayLineR;` (one per channel, written once per sample, tapped multiple times)
- Per-tap lowpass filters: `signalsmith::filters::BiquadStatic<float> tapFiltersL[8], tapFiltersR[8];`
- SmoothedValue members: `roomSizeSmoothed` (50ms ramp), `proximitySmoothed` (20ms ramp), `widthSmoothed` (20ms ramp), `bypassBlend` (10ms ramp)

**ReflectionsSection.cpp -- Full implementation:**

1. **7 Shape Presets** -- Use the exact tap pattern data from 03-RESEARCH.md Code Examples section (The Parlour, The Gallery, The Chamber, The Nave, The Alcove, The Crypt, The Conservatory). Copy the complete `kShapes` array verbatim from the research.

2. **Room Size Mapping Function** (private helper):
   ```cpp
   // Maps normalized 0-1 knob value to delay multiplier
   // The NormalisableRange skew handles the UI weighting; here we map to ms
   // Room Size 0.0 -> 1ms base delay, Room Size 1.0 -> 30ms base delay
   static float roomSizeToDelayMs(float normalized)
   {
       return 1.0f + normalized * 29.0f;  // Linear 1-30ms
       // The perceptual weighting is already in the NormalisableRange skew
   }
   ```

3. **prepare():**
   - Set sample rate and block size
   - Size delay lines for max 35ms (30ms + margin): `maxSamples = ceil(35.0 * sampleRate / 1000.0)`
   - `delayLineL.resize(maxSamples + 1); delayLineL.reset();` (same for R)
   - Reset all 16 tap filters
   - Initialize SmoothedValues with ramp times: roomSize 50ms, proximity 20ms, width 20ms, bypassBlend 10ms
   - Set bypassBlend current and target based on bypassed state

4. **process() -- Per-sample processing:**
   ```
   For each sample s:
     a. Read smoothed values: roomSize, proximity, width, blend
     b. Compute delay multiplier: delayMs = roomSizeToDelayMs(roomSize)
     c. Write input samples to delay buffers:
        delayLineL.write(inputL[s])
        delayLineR.write(inputR[s])
     d. Read all 8 L taps and 8 R taps from current shape:
        For each tap t:
          - lDelayMs = lerp(monoDelay, shapeDelayL[t], width) * delayMs / 30.0f
            (where monoDelay = (shapeDelayL[t] + shapeDelayR[t]) * 0.5f)
          - Scale delay to samples: lDelaySamples = lDelayMs * sampleRate / 1000.0f
          - Read from delay: tapVal = delayLineL.read(lDelaySamples)
          - Apply per-tap lowpass filter (cutoff = 12000 - absorptionRate * lDelayMs * 200)
            clamped to [1000, 16000] Hz, scale by sampleRate
          - tapVal = tapFiltersL[t](tapVal)
          - Apply gain: tapVal *= gainL[t]
          - Accumulate into sumL and sumR weighted by pan position
          (Repeat for R taps reading from delayLineR)
     e. Apply proximity blend:
        - directGain = lerp(1.0f, pow(10, -12/20), proximity)  // 0dB to -12dB
        - reflGain = lerp(pow(10, -18/20), 1.0f, proximity)    // -18dB to 0dB
        - outputL = inputL * directGain + sumL * reflGain
        - outputR = inputR * directGain + sumR * reflGain
     f. Apply bypass crossfade:
        - finalL = inputL * (1.0 - blend) + outputL * blend
        - finalR = inputR * (1.0 - blend) + outputR * blend
     g. Write output
   ```

5. **Shape crossfade:** When `setShape()` is called with a new index:
   - Store new index as `pendingShapeIndex`
   - In process(), when pendingShapeIndex != -1, run both old and new shape tap patterns simultaneously and crossfade over ~30ms (use a simple linear ramp counter). When crossfade completes, set `currentShapeIndex = pendingShapeIndex; pendingShapeIndex = -1;`

6. **Subtle room size darkening** (Claude's Discretion): As room size increases, reduce per-tap filter cutoff. Use formula: `baseCutoff = 12000.0f - roomSize * 6000.0f` (12kHz at min size, 6kHz at max size). This provides physically-motivated darkening while keeping the Air stage as primary tonal control.

7. **Width implementation:** Width 0% makes L and R taps use averaged (mono) delay times/gains/pans. Width 100% uses the full preset-defined L/R decorrelation. Interpolate linearly between mono and stereo tap configurations per the research code example.

8. **setRoomSize(), setShape(), setProximity(), setWidth(), setBypass()** -- all follow the SmoothedValue pattern from ResonanceSection. setShape does the crossfade setup. Others just call setTargetValue.

9. **reset()** -- Clear delay lines, reset all filters, reset SmoothedValues.

**Important implementation details:**
- Do NOT couple Stage I material to reflections (per discretion recommendation)
- Size and Shape are independent (any shape works at any size)
- Pan position for each tap distributes energy between L and R output using equal-power pan law: `panL = cos(pan * PI/4 + PI/4), panR = sin(pan * PI/4 + PI/4)` or simpler linear pan
- Safety: clamp all delay reads to >= 0.0f samples
- Normalization: sum of tap gains per channel should be approximately 1.0 (the preset values are already normalized)
