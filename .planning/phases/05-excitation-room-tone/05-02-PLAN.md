```cpp
struct KellettPinkNoise
{
    float b0 = 0.0f, b1 = 0.0f, b2 = 0.0f, b3 = 0.0f;
    float b4 = 0.0f, b5 = 0.0f, b6 = 0.0f;

    float process(float white)
    {
        b0 = 0.99886f * b0 + white * 0.0555179f;
        b1 = 0.99332f * b1 + white * 0.0750759f;
        b2 = 0.96900f * b2 + white * 0.1538520f;
        b3 = 0.86650f * b3 + white * 0.3104856f;
        b4 = 0.55000f * b4 + white * 0.5329522f;
        b5 = -0.7616f * b5 - white * 0.0168980f;
        float pink = b0 + b1 + b2 + b3 + b4 + b5 + b6 + white * 0.5362f;
        b6 = white * 0.115926f;
        return pink * 0.11f;
    }

    void reset() { b0 = b1 = b2 = b3 = b4 = b5 = b6 = 0.0f; }
};
```

**RoomToneSection.cpp** -- Full implementation:

`prepare()`:
- Store sampleRate, blockSize
- Initialize rngL and rngR with different seeds (constructor or setSeed)
- Reset Kellett filters
- Set up spectral shaping filters at default Room Size/Shape: call updateShapingFilters()
- Reset SmoothedValues:
  - `ambienceSmoothed.reset(sampleRate, 0.025)` -- 25ms
  - `bypassBlend.reset(sampleRate, 0.010)` -- 10ms
  - `gateEnvelope.reset(sampleRate, 0.050)` -- start with 50ms ramp (will be asymmetric in process)
- Set initial values: `ambienceSmoothed.setCurrentAndTargetValue(0.1)`, `bypassBlend.setCurrentAndTargetValue(0.0f)` (default bypassed)
- Reset LFO phase to 0

`updateShapingFilters()`:
Room Size (0.0-1.0) shifts filter parameters:
- HPF cutoff: 100Hz at roomSize=0.0 -> 60Hz at roomSize=1.0: `hpfFreq = 100.0f - 40.0f * currentRoomSize`
- Presence peak freq: 400Hz at small -> 200Hz at large: `peakFreq = 400.0f - 200.0f * currentRoomSize`
- Presence peak gain: +4dB base
- LPF cutoff: 10000Hz at small -> 6000Hz at large: `lpfFreq = 10000.0f - 4000.0f * currentRoomSize`

Apply Shape offsets from kShapeAmbience[currentShapeIndex]:
- `peakFreq += shapeChar.presenceFreqOffset` (clamped to 100-800Hz)
- `lpfFreq += shapeChar.lpfCutoffOffset` (clamped to 4000-12000Hz)
- `peakGainDb = 4.0f + shapeChar.presenceGainDb` (clamped to 1-7dB)

Configure BiquadStatic filters using Signalsmith API:
- `hpfL.highpassQ(hpfFreq / sampleRate, 0.707f)` (same for R)
- `presenceL.peakDb(peakFreq / sampleRate, 0.8f, peakGainDb)` (same for R)
- `lpfL.lowpassQ(lpfFreq / sampleRate, 0.707f)` (same for R)

Note: Signalsmith BiquadStatic takes normalized frequency (freq/sampleRate), not Hz.

`process()`:
1. Early exit checks (same pattern as AirSection).
2. If bypassed and not smoothing, return immediately. When bypassed, skip all noise generation for CPU savings.
3. Per-sample loop:
```
for each sample s:
    blend = bypassBlend.getNextValue()
    ambience = ambienceSmoothed.getNextValue()

    // Compute gate multiplier
    float gate = 1.0f
    if (gateMode == TransportOnly && !transportPlaying):
        gateEnvelope.setTargetValue(0.0f)
        gate = gateEnvelope.getNextValue()
    elif (gateMode == SignalGated):
        // Measure input level from current sample (simple per-sample envelope)
        float inputPower = (channelL[s]*channelL[s])
        if channelR: inputPower = (inputPower + channelR[s]*channelR[s]) * 0.5f
        float inputDb = Decibels::gainToDecibels(std::sqrt(inputPower), -100.0f)
        if (inputDb > kGateThresholdDb):
            gateEnvelope.setTargetValue(1.0f)
        else:
            gateEnvelope.setTargetValue(0.0f)
        gate = gateEnvelope.getNextValue()
    else: // AlwaysOn
        gateEnvelope.setTargetValue(1.0f)
        gate = gateEnvelope.getNextValue()

    // Generate decorrelated white noise
    float whiteL = rngL.nextFloat() * 2.0f - 1.0f
    float whiteR = rngR.nextFloat() * 2.0f - 1.0f

    // Pink via Kellett IIR
    float pinkSampleL = pinkL.process(whiteL)
    float pinkSampleR = pinkR.process(whiteR)

    // Spectral shaping: HPF -> Presence -> LPF
    float shapedL = lpfL(presenceL(hpfL(pinkSampleL)))
    float shapedR = lpfR(presenceR(hpfR(pinkSampleR)))

    // Slow LFO modulation for natural quality
    float lfoValue = std::sin(lfoPhase * 2.0f * MathConstants<float>::pi)
    float modGain = Decibels::decibelsToGain(lfoValue * kLfoDepthDb, -100.0f)
    lfoPhase += kLfoRateHz / (float)currentSampleRate
    if (lfoPhase >= 1.0f) lfoPhase -= 1.0f

    // Level calculation: map ambience 0-1 to gain
    // At ambience=0: silence. At ambience=1: kMaxNoiseGainDb (-35dB)
    // Use a curve that gives useful range across the knob
    float noiseGain = (ambience > 0.001f)
        ? Decibels::decibelsToGain(kMaxNoiseGainDb + (1.0f - ambience) * (-60.0f - kMaxNoiseGainDb), -100.0f)
        : 0.0f
    // This maps: ambience=0 -> -inf (silence), ambience=0.5 -> ~-47.5dB, ambience=1.0 -> -35dB

    float totalGain = noiseGain * gate * modGain * blend

    // ADDITIVE: noise added to signal (not replacing)
    channelL[s] += shapedL * totalGain
    if (channelR) channelR[s] += shapedR * totalGain
```

For the Signal-Gated mode, the gateEnvelope SmoothedValue needs asymmetric attack/release. Since SmoothedValue has a single ramp time, handle this by setting different ramp times based on direction:
- When target goes to 1.0 (opening): use attack time (50ms)
- When target goes to 0.0 (closing): use release time (500ms)

Approach: track previous gate target. When it changes, call `gateEnvelope.reset(sampleRate, newTimeInSeconds)` then `setTargetValue()`. This gives asymmetric behavior.

Actually, simpler: use a manual one-pole envelope follower for the gate instead of SmoothedValue. Store `gateLevel` float, compute attack/release coefficients in prepare():
```
gateAttackCoeff = 1.0f - std::exp(-1.0f / (sampleRate * kGateAttackMs * 0.001f))
gateReleaseCoeff = 1.0f - std::exp(-1.0f / (sampleRate * kGateReleaseMs * 0.001f))
```
Then per sample: `float coeff = (target > gateLevel) ? gateAttackCoeff : gateReleaseCoeff; gateLevel += coeff * (target - gateLevel);`

Use this manual approach. Replace the SmoothedValue gateEnvelope with `float gateLevel = 0.0f` and `float gateAttackCoeff, gateReleaseCoeff`.

`reset()`:
- Reset Kellett filters, biquad filters, RNG (re-seed), SmoothedValues, LFO phase, gateLevel

`setBypass()`:
- `bypassBlend.setTargetValue(shouldBypass ? 0.0f : 1.0f)`

Other setters: store values, call `updateShapingFilters()` when Room Size or Shape changes.

IMPORTANT: No allocations in process(). Noise generation is purely computational.
IMPORTANT: When bypassed, skip noise generation entirely for CPU savings (TONE-07).
IMPORTANT: Level is purely Ambience knob -- NO coupling to Proximity or other params (locked decision).
